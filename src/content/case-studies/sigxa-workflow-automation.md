## Project Overview

SigXA needed a dependable lead workflow that could run every day without manual coordination. We designed and implemented a deterministic automation system across Apollo.io and n8n so lead routing, outreach execution, and activity tracking follow a predictable path.

The goal was operational consistency: reduce ambiguity in follow-up actions, keep sequence logic controlled, and ensure each stage updates source-of-truth records cleanly.

## The Challenge

The outreach workflow had to coordinate multiple systems while staying reliable under daily execution. The team needed a setup that could:

- Run scheduled lead actions with clear, deterministic branching
- Personalize outreach while preserving control over message quality and timing
- Keep CRM and operational records synchronized after each action
- Avoid brittle automations that require constant intervention

A second challenge was evaluating newer agentic tooling without risking production stability.

## Our Solution

We implemented a deterministic automation architecture with clear ownership per platform:

- **Apollo.io (CRM + Sequencing)**: Configured physician outreach sequences and operational controls for campaign execution, contact progression, and sequence-level management. Email scheduling and follow-up cadences run automatically on a defined schedule — no manual coordination required.
- **n8n + Zapier (Workflow Orchestration)**: Built the backend orchestration pipeline for scheduled lead processing, action routing, conditional email handling, Google Sheets updates, and execution tracking across systems.
- **Claude AI (Personalized Message Generation)**: Integrated Claude to generate clinic-specific outreach messages based on market research pulled via clinic API integration. Each message is tailored to the clinic's context — specialty, location, patient volume signals — rather than using generic templates.
- **Openclaw (Experimental Track)**: Evaluated for basic task management and exploratory control flows. Useful for rapid prototyping, but Apollo.io + n8n remained the production path for reliability and control.

## Outcome

The automation eliminated the equivalent of one full SDR's manual workload. Email scheduling, follow-up sequencing, and CRM record updates now run without human intervention across the entire outreach pipeline.

- **1 FTE equivalent** of SDR work replaced by automated workflows
- **100% of email scheduling** handled through structured automation — no manual sending or calendar management
- **AI-personalized outreach** per clinic, generated by Claude based on real market research data from clinic API integration
- **5+ platforms orchestrated** end-to-end: Apollo.io, n8n, Zapier, Google Sheets, Gmail, and clinic data sources

The implementation also established a clear tooling boundary: production workflows run on the controllable Apollo.io + n8n + Zapier stack, while emerging platforms can be evaluated without risking core operations.
